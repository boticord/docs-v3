# Введение
Любой желающий может внести свой вклад в развитие проекта. Для этого необходимо следовать следующим правилам:

## Основное
1. Все изменения должны быть четко описаны в коммите и pull request'е.
2. Ведите себя прилично. Не используйте нецензурную лексику и не оскорбляйте других участников проекта.
3. Желательно пользоваться английским языком, поскольку является международным языком

## Ветки
- `main` - основная ветка проекта
- `dev` - ветка разработки

## Коммиты
Четких правил для коммитов нет, но есть несколько рекомендаций:
- Комиты должны желательно быть на английском языке
- Использовать [conventional commits](https://conventionalcommits.org)

### Примеры
1. <strike>Changed text</strike> ❌ Коммит не описывает, почему изменен текст, где.
3. **chore(Faq): Added additional answer to the question ** ✅ Идеально!


## Документация
Документация должна быть написана на русском языке. Для написания документации используется [Docusaurus](https://docusaurus.io/).
Если вы хотите изменить документацию на другом языке, то используйте [Crowin](https://crowdin.com/).

## Я разработчик библиотеки для вашего API, как мне добавить её в вашу документацию?
Чтобы это сделать, создайте форк репозитория, добавьте нужную документацию и отправьте пулл реквест в наш репозиторий используя [форматирование](https://docusaurus.io/docs/markdown-features) markdown.
Шаблон для добавления информации о библиотеке:
```md
---
title: Имя
description: Описание
slug: /sdk/имя
---

<описание библиотеки>
```
Форма pull request'а:
```md
Добавлена информация о библиотеке <название библиотеки>

Язык: <язык программирования>
Библиотека: <название библиотеки>

<описание библиотеки>

Ссылка на репозиторий: <ссылка на репозиторий>

Ссылка на документацию: <ссылка на документацию>

Ссылка на автора: <ссылка на автора>
```


## Перевод
Для перевода мы используем [Crowdin](https://crowdin.com/). Если вы хотите помочь с переводом, то перейдите по [ссылке](https://crowdin.com/).
Изменения файлов в папке `locale` не допускаются.
